# Сеть кластера: приватная сеть (РФ) + AmneziaWG (KZ)

Кластер из 6 узлов связывается двумя способами, по зонам:

- **5 РФ-узлов** (`node-s1/s2/s3/heavy/dev`) - приватная сеть провайдера. Быстро, бесплатно, DPI не
  касается (трафик не выходит в публичный интернет). k3s едет по приватным IP.
- **node-eu (KZ)** - через границу по **AmneziaWG** (WireGuard с обфускацией против DPI). Подключается
  "звездой": отдельный точка-точка туннель к каждому из 5 РФ-узлов. Через границу идёт только
  обфусцированный awg-трафик llm-пода; WireGuard-сигнатуры в нём нет, ТСПУ его не палит.

```
        приватная сеть провайдера (10.16.0.0/24, vxlan, без шифрования)
   node-s1 ── node-s2 ── node-s3 ── node-heavy ── node-dev
      │          │          │           │            │
      └──────────┴────── AmneziaWG (awg0, обфускация) ┴──────┐
                                                            node-eu (KZ)
```

## Почему так, а не иначе

- **Не полный amnezia-mesh.** Обфускация РФ↔РФ - лишний CPU/латентность там, где DPI нет. Приватная
  сеть для пятёрки быстрее и проще.
- **Не один хаб KZ↔s1.** flannel vxlan гоняет pod-to-pod напрямую узел-узел. backend (data-узлы) и
  staging-backend (`node-dev`) оба зовут llm на KZ, значит KZ должен напрямую видеть и data-узлы, и
  `node-dev`. Хаб через s1 сделал бы его SPOF и роутером всего llm-трафика. Поэтому звезда KZ↔все 5.

## Предусловия

1. Все 5 РФ-узлов в **одной** приватной подсети `10.16.0.0/24`, интерфейс `ens9`. Проверь:
   `ip -br a` (виден `ens9` с адресом `10.16.0.x`), `ping <приватный IP соседа>`.
2. AmneziaWG установлен на **всех 6** узлах (РФ-узлы тоже - они держат awg-линк к KZ). Ubuntu:
   ```bash
   sudo add-apt-repository -y ppa:amnezia/ppa
   sudo apt update && sudo apt install -y amneziawg amneziawg-tools
   ```
   (`amneziawg` - DKMS-модуль, нужны `linux-headers-$(uname -r)`; на нестандартном ядре - пакет
   `amneziawg-go` как userspace-замена.)

## Шаги

1. Все IP в [`gen-awg-eu.sh`](gen-awg-eu.sh) уже проставлены (РФ-подсеть `10.16.0.0/24`, KZ-tunnel
   `10.10.0.6`). При необходимости поменяй параметры обфускации `Jc..H4` на свои - главное, одинаковые
   на всех узлах.
2. Сгенерируй конфиги (на машине с `amneziawg-tools`):
   ```bash
   cd deploy/amnezia && ./gen-awg-eu.sh
   ```
   Появится `out/<node>/awg0.conf` на каждый из 6 узлов (ключи генерятся внутри, наружу не светятся).
3. Разложи и подними на каждом узле:
   ```bash
   scp out/<node>/awg0.conf <node>:/etc/amnezia/amneziawg/awg0.conf
   ssh <node> 'awg-quick up awg0 && systemctl enable awg-quick@awg0'
   ```
4. Проверь связность через границу - с `node-eu`:
   ```bash
   awg show                       # handshake со всеми 5 РФ-узлами, растущие rx/tx
   ping <приватный IP node-s1>    # идёт через awg0
   ```

## Firewall

- **Между публичными IP** (KZ ↔ каждый РФ-узел): только `51820/udp` (AmneziaWG).
- **В приватной сети** (РФ↔РФ): порты k3s `6443/tcp`, `2379-2380/tcp` (server-узлы), `8472/udp`
  (flannel vxlan), `10250/tcp` - на приватном интерфейсе.
- **Наружу публично**: `80/443` только на data-узлах, куда смотрит DNS.

Служебные порты k3s наружу не открываются: РФ↔РФ они в приватной сети, KZ↔РФ - внутри awg-туннеля.
